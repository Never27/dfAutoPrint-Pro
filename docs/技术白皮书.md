## 1. 摘要

PdfAutoPrint Pro 是一款面向 Windows 平台的虚拟 PDF 打印机管理解决方案。该系统通过实时监控 Windows 打印池目录，自动捕获打印任务并借助 GhostScript 引擎将 PostScript 文件转换为高质量 PDF 文档。

### 核心优势

| 特性 | 优势 |
|------|------|
| **全自动化** | 无需人工干预，打印即自动生成 PDF |
| **零 NuGet 依赖** | 纯 .NET 内置 API 实现，部署简单、稳定可靠 |
| **PostScript 级水印** | 水印直接注入 PS 代码，转换后不可篡改 |
| **灵活配置** | 多打印机配置独立管理，支持文件名模板 |
| **轻量架构** | JSON 文件存储，无需数据库引擎 |

### 技术指标

- **运行时**: .NET 8 (LTS)
- **目标平台**: Windows 10/11 (x64)
- **启动时间**: < 2 秒
- **内存占用**: < 100 MB (空闲时)
- **外部依赖**: 仅 GhostScript CLI

## 3. 核心组件详解

### 3.1 打印机监控组件 (SpoolMonitor)

#### 功能描述

SpoolMonitor 是整个系统的入口组件，负责实时监控 Windows 打印池目录，检测新产生的 `.prn` (PostScript) 文件并触发处理管线。

#### 技术实现

- **文件系统监控**: 基于 `System.IO.FileSystemWatcher`，监控打印池目录的 `Created` 事件
- **文件就绪检测**: 通过文件独占访问尝试判断文件是否写入完成
- **并发控制**: 使用 `SemaphoreSlim` 限制同时处理的作业数量，防止资源耗尽
- **错误隔离**: 每个作业在独立的 try-catch 块中处理，单个失败不影响整体监控

#### 监控流程

```
1. 启动 FileSystemWatcher，监听 .prn 文件创建事件
2. 检测到新文件 → 等待文件锁释放 (轮询重试)
3. 获取信号量许可 → 进入处理管线
4. 处理完成 → 释放信号量 → 等待下一个文件
```

#### 关键参数

| 参数 | 默认值 | 说明 |
|------|--------|------|
| 监控文件类型 | `*.prn` | PostScript 打印文件 |
| 文件就绪重试次数 | 10 次 | 超过则视为异常 |
| 文件就绪重试间隔 | 500ms | 每次重试间隔 |
| 最大并发数 | 3 | 同时处理的作业数 |

### 3.2 PDF 转换引擎 (GhostScriptService)

#### 功能描述

GhostScriptService 封装了 GhostScript 命令行工具的调用逻辑，将 PostScript 文件转换为 PDF 格式。

#### 技术实现

- **进程管理**: 通过 `System.Diagnostics.Process` 启动 `gswin64c.exe`
- **异步通信**: 使用 `RedirectStandardOutput/Error` 异步读取进程输出
- **超时控制**: 支持转换超时设定，超时后强制终止进程
- **参数构建**: 根据 PDF 质量配置动态构建 GhostScript 命令行参数

#### 转换参数映射

| PDF 质量等级 | GhostScript 参数 | 用途 |
|-------------|-----------------|------|
| `screen` (屏幕) | `-dPDFSETTINGS=/screen` | 72 DPI, 适合屏幕阅读 |
| `ebook` (电子书) | `-dPDFSETTINGS=/ebook` | 150 DPI, 适合电子阅读器 |
| `printer` (打印) | `-dPDFSETTINGS=/printer` | 300 DPI, 适合打印输出 |
| `prepress` (印刷) | `-dPDFSETTINGS=/prepress` | 300+ DPI, CMYK, 适合印刷 |

#### 进程生命周期管理

```
启动 gswin64c.exe
      │
      ▼
异步读取 stdout / stderr (防止死锁)
      │
      ▼
等待进程退出 (带超时)
      │
      ├─ 正常退出 (ExitCode = 0) ──▶ 转换成功
      │
      └─ 异常退出 (ExitCode ≠ 0) ──▶ 抛出 GsException
                                          │
                                          ▼
                                    记录错误日志
                                    更新作业状态为 Failed
```

### 3.3 水印注入引擎 (PsWatermarkEngine)

#### 功能描述

PsWatermarkEngine 实现了基于 PostScript 代码注入的水印方案。水印代码直接嵌入到 PostScript 文件中，在 GhostScript 转换阶段渲染为 PDF 页面的一部分，具有不可篡改性。

#### 技术优势

与传统 PDF 层水印相比，PostScript 级水印有以下优势:

| 对比维度 | PDF 层水印 | PostScript 注入水印 |
|----------|-----------|-------------------|
| 可移除性 | 可被 PDF 工具移除 | 渲染到页面内容中，不可分离 |
| 渲染质量 | 依赖 PDF 工具 | GhostScript 矢量渲染，质量更高 |
| 性能影响 | 需要后处理步骤 | 转换时一并完成，无额外开销 |
| 灵活性 | 有限 | 完全控制 PS 图形状态 |

#### 支持的水印类型

```
水印类型
├── 文字水印 (TextWatermark)
│   ├── 自定义文本内容
│   ├── 字体与字号
│   ├── 颜色 (RGB)
│   ├── 旋转角度
│   ├── 透明度
│   └── 位置 (X, Y)
│
├── 图片印章 (ImageWatermark)
│   ├── 图片文件路径
│   ├── 位置与缩放
│   └── 透明度
│
└── 页面叠加 (PageOverlay)
    ├── 页眉 (Header)
    ├── 页脚 (Footer)
    └── 页码 (Page Number)
```

#### 注入流程

```
原始 .prn 文件
      │
      ▼
解析 PostScript 页面结构
      │
      ▼
定位页面渲染结束位置
      │
      ▼
在每页渲染结束前插入:
  ┌─────────────────────┐
  │ gsave               │  保存当前图形状态
  │ [水印 PS 代码]       │  执行水印渲染
  │ grestore            │  恢复图形状态
  └─────────────────────┘
      │
      ▼
输出注入后的 .prn 文件 → 送入 GS 转换
```

### 3.4 配置管理系统 (ConfigService)

#### 功能描述

ConfigService 提供基于 JSON 文件的配置管理方案，使用 `System.Text.Json` 实现序列化和反序列化。

#### 存储方案

| 存储项 | 文件路径 | 说明 |
|--------|----------|------|
| 应用配置 | `config.json` | 全局设置、GS路径、默认参数 |
| 打印机配置 | `printer-profiles.json` | 所有打印机配置集合 |
| 作业历史 | `job-history.json` | 打印作业记录 |
| 日志文件 | `logs/app-{date}.log` | 按日期轮转的日志 |

#### 数据持久化策略

```
配置读写策略:
  ├── 读取: 启动时一次性加载到内存
  ├── 修改: 内存中修改后立即写入文件
  ├── 备份: 每次写入前创建 .bak 备份
  └── 恢复: 主文件损坏时从 .bak 恢复

作业历史策略:
  ├── 追加: 新记录追加到 JSON 数组
  ├── 查询: 内存中 LINQ 过滤
  ├── 清理: 按配置天数自动清理旧记录
  └── 导出: 支持导出为 CSV 格式
```

## 4. 安全考量

### 4.1 进程安全

| 风险点 | 防护措施 |
|--------|----------|
| GhostScript 参数注入 | 参数白名单校验，不直接拼接用户输入 |
| PowerShell 命令注入 | 使用 PowerShell AST 参数传递，避免字符串拼接 |
| 进程权限提升 | GS 以普通用户权限运行，使用 `-dSAFER` 安全模式 |
| 进程泄漏 | `using` 语句确保进程资源释放，超时强制终止 |

### 4.2 文件系统安全

| 风险点 | 防护措施 |
|--------|----------|
| 文件路径遍历 | 校验输出路径在配置的目录范围内 |
| 文件权限 | 检查目录读写权限后再执行操作 |
| 敏感信息泄露 | 日志中不记录完整文件内容，仅记录路径 |
| 临时文件清理 | 处理完成后删除中间 .prn 文件 |

### 4.3 数据安全

| 风险点 | 防护措施 |
|--------|----------|
| 配置文件损坏 | 写入前创建 .bak 备份文件 |
| JSON 解析异常 | 容错处理，异常时恢复默认配置 |
| 并发写入冲突 | 配置写入使用文件锁机制 |

### 4.4 GhostScript 安全模式

```
-dSAFER 参数启用 GS 安全模式:
  ├── 禁止写入任意文件
  ├── 禁止执行系统命令
  ├── 限制内存使用
  └── 限制文件访问路径
```

### 6.2 一键安装脚本

`install.ps1` 脚本执行以下步骤:

```
1. 检查 .NET 8 Runtime 是否已安装
   └── 未安装 → 下载并安装

2. 检查 GhostScript 是否已安装
   └── 未安装 → 提示用户安装

3. 创建虚拟打印机端口
   └── PowerShell: Add-PrinterPort

4. 安装虚拟打印机驱动
   └── PowerShell: Add-PrinterDriver

5. 创建虚拟打印机
   └── PowerShell: Add-Printer

6. 复制应用程序文件到安装目录
   └── 默认: C:\Program Files\PdfAutoPrint Pro\

7. 创建桌面快捷方式

8. 配置自动启动 (可选)

9. 注册文件关联 (可选)
```

### 6.3 单文件发布

```bash
dotnet publish -c Release -r win-x64 \
    --self-contained \
    -p:PublishSingleFile=true \
    -p:IncludeNativeLibrariesForSelfExtract=true \
    -p:CompressionLevel=Maximum
```

发布产物: 单个 `PdfAutoPrint.Pro.exe` 文件，无需安装 .NET Runtime。
