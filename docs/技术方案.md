## 1. 问题定义

### 1.1 业务背景

在日常办公和文档处理场景中，用户需要将各种格式的文档转换为 PDF。虽然 Windows 10+ 内置了 "Microsoft Print to PDF" 打印机，但功能极其有限：
- 无法自定义文件名（每次弹窗输入）
- 无法添加水印
- 无法批量处理
- 无任务历史记录
- 无自动分类存储

### 1.2 目标

构建一个功能完整的虚拟 PDF 打印解决方案，提供自动化、可配置、可追溯的 PDF 输出体验。

## 3. 技术选型

### 3.1 为什么 .NET 8 WPF？

| 选项 | 评价 |
|------|------|
| .NET 8 WPF | ✅ 原生 Windows 支持，性能好，开发效率高 |
| Electron | ❌ 资源占用大，不符合"轻量级"要求 |
| WinUI 3 | ❌ 需要 Windows 10 1809+，不支持 Win7 |
| WinForms | ❌ UI 开发效率低，不支持现代布局 |

### 3.2 为什么零 NuGet 依赖？

- .NET 10 SDK 的 NuGet 还原存在兼容性问题（`Path.Combine(null, ...)` 异常）
- 项目功能完全可用 .NET 内置 API 实现
- 减少部署复杂度，无需考虑依赖版本冲突
- JSON 替代 SQLite 满足配置存储需求

### 3.3 为什么 JSON 而非 SQLite？

| 特性 | JSON | SQLite |
|------|------|--------|
| 依赖 | 无 | 需 System.Data.SQLite NuGet |
| 可读性 | 人工可读 | 需工具查看 |
| 性能 | 小数据量足够 | 大数据量更快 |
| 备份 | 直接复制 | 需导出 |

## 4. 核心模块设计

### 4.1 打印队列监控 (SpoolMonitor)

```
FileSystemWatcher → Created/Changed 事件 → 防抖等待(500ms)
    → 读取 PS 文件 → 提取元数据 → [水印注入] → GS 转换 → 保存 → 记录
```

**关键技术点**：
- `FileSystemWatcher` 监听 spool 目录
- 防抖机制：`WaitForFileReadyAsync()` 轮询确认文件大小稳定
- 元数据提取：正则匹配 PS 头部 `%%Title:`, `%%For:` 等 DSC 注释

### 4.2 GhostScript 集成 (GhostScriptService)

```
Process.Start("gswin64c.exe", arguments) → 等待退出 → 检查输出文件
```

**参数构建**：
- 基础：`-dNOPAUSE -dBATCH -sDEVICE=pdfwrite -sOutputFile=...`
- 灰度：`-sColorConversionStrategy=Gray -dProcessColorModel=/DeviceGray`
- 图片模式：`-r<DPI> -dHaveTransparency=false`
- 原始模式：`-dPDFSETTINGS=/printer -dEmbedAllFonts=true`

### 4.3 水印注入引擎 (PsWatermarkEngine)

```
原始 PS 文本 → 找到 %%Page: 或 %%EndSetup 注入点 → 插入水印 PS 代码 → 输出
```

**水印 PS 代码示例**：
```
gsave
/FontName findfont 48 scalefont setfont
1 0 0 setrgbcolor
0.3 .setopacityalpha
300 400 moveto
45 rotate
(机密文件) show
grestore
```

### 4.4 打印机管理 (PrinterManager)

通过 PowerShell 命令管理打印机，无需 System.Management：

```powershell
# 获取打印机
Get-WmiObject -Class Win32_Printer | Select-Object Name, DriverName, PortName

# 安装打印机
Add-PrinterPort -Name $portName
Add-Printer -Name $name -DriverName $driverName -PortName $portName

# 设置默认打印机
(New-Object -ComObject WScript.Network).SetDefaultPrinter($name)

# 重启 Spooler
Restart-Service -Name Spooler
```

## 5. 数据模型

### 核心实体关系

```
GlobalConfig
├── List<PrinterProfile>
│   ├── OutputConfig (自动保存、输出路径、监听目录)
│   ├── FileNameTemplate (文件名模式、占位符规则)
│   ├── ConflictConfig (冲突策略、递增连接符)
│   ├── QualityConfig (色彩模式、输出模式、DPI)
│   ├── NotificationConfig (系统托盘、成功/失败通知)
│   ├── ServiceConfig (失败跳过、GS路径、开机自启)
│   └── WatermarkConfig
│       └── List<WatermarkItem>
│           ├── WatermarkType (Text/Image)
│           ├── WatermarkPosition (锚点、偏移、单位)
│           ├── PageRange (跳过页面)
│           └── FontConfig (字体、颜色)
└── LogConfig (保留天数、日志级别)

JobRecord (独立 JSON 文件)
├── PrinterName, JobName, Author
├── SourceFile, OutputFile, FileSize, PageCount
├── Status, ErrorMessage
└── StartTime, EndTime, DurationMs
```

## 6. 错误处理策略

| 层级 | 策略 |
|------|------|
| 文件读取失败 | 如果 SkipOnFailure=true → 跳过记录 |
| GS 转换失败 | 日志错误 + 任务标记失败 |
| 冲突处理失败 | 回退到递增编号策略 |
| 配置读写失败 | 使用默认配置 + 日志告警 |
| PowerShell 执行失败 | 超时重试 + 友好错误提示 |

---

## 7. 性能考虑

- **文件监控**：单线程处理，避免并发冲突
- **JSON 读写**：使用 `System.Text.Json` 高性能序列化
- **GS 转换**：异步 Process.Start，不阻塞 UI
- **日志大小**：保留天数策略防止日志膨胀
- **任务历史**：最多保留 1000 条记录
