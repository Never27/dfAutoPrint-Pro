## 1. 项目概述

### 1.1 项目目标

PdfAutoPrint Pro 是一款 Windows 平台上的虚拟 PDF 打印机管理器，基于 WPF 桌面应用框架构建。其核心目标是：

- **自动化 PDF 生成**: 监控 Windows 打印池，自动将打印任务转换为高质量 PDF 文件
- **灵活的打印机管理**: 支持创建、配置和管理多个虚拟 PDF 打印机配置
- **水印与页面装饰**: 支持 PostScript 级别的文字水印、图片印章、页眉页脚等
- **零外部依赖**: 不依赖任何 NuGet 第三方包，完全基于 .NET 内置 API
- **轻量级部署**: 单文件发布，一键安装

### 1.2 核心特性

| 特性 | 说明 |
|------|------|
| 打印池监控 | 基于 FileSystemWatcher 实时监控打印机池目录 |
| PS→PDF 转换 | 通过 GhostScript CLI 将 PostScript 文件转换为 PDF |
| 水印注入 | 通过 PostScript 代码注入实现文字/图片水印 |
| 多打印机配置 | 支持创建多个打印机配置，独立管理输出参数 |
| 文件名模板 | 支持日期、序号、自定义占位符的文件名模板 |
| 冲突处理 | 支持覆盖、递增、备份三种文件冲突处理策略 |
| 作业历史 | JSON 格式的完整打印作业历史记录 |
| 日志系统 | 文件+控制台双输出，支持日志保留策略 |

## 2. 技术栈

| 技术 | 版本 | 用途 |
|------|------|------|
| .NET | 8.0 | 运行时框架 |
| WPF | .NET 8 内置 | 桌面 UI 框架 |
| C# | 12.0 | 编程语言 |
| MVVM | 手动实现 | UI 架构模式 |
| GhostScript | 10.x (外部) | PS→PDF 转换引擎 |
| JSON | System.Text.Json | 数据序列化 |
| PowerShell/WMI | 系统内置 | 打印机管理 |

### 关键技术决策

**零 NuGet 依赖策略**:
- .NET 10 SDK 存在 NuGet 包还原问题，因此项目采用纯 .NET 内置 API
- `System.Text.Json` 替代 Newtonsoft.Json
- PowerShell 进程调用替代 System.Management NuGet 包
- JSON 文件存储替代 SQLite

## 3. 开发环境搭建

### 3.1 系统要求

- **操作系统**: Windows 10/11 (x64)
- **.NET SDK**: 8.0.x
- **IDE**: Visual Studio 2022 (17.8+) 或 JetBrains Rider 2023.3+
- **GhostScript**: 10.x (用于 PS→PDF 转换测试)

### 3.2 安装步骤

1. 安装 .NET 8 SDK: [https://dotnet.microsoft.com/download/dotnet/8.0](https://dotnet.microsoft.com/download/dotnet/8.0)

2. 克隆项目:
```bash
git clone <repository-url>
cd PdfAutoPrint.Pro
```

3. 安装 GhostScript (开发时需要):
   - 下载 gswin64c.exe 并安装
   - 确认 `gswin64c.exe` 在系统 PATH 中，或配置路径到 `appsettings.json`

### 3.3 项目配置

项目配置文件位于 `%LocalAppData%/PdfAutoPrint.Pro/` 目录下:

```
%LocalAppData%/PdfAutoPrint.Pro/
├── config.json          # 主配置文件
├── printer-profiles.json # 打印机配置集合
├── job-history.json      # 作业历史记录
└── logs/                 # 日志文件目录
```

## 5. 构建与运行

## 6. 编码规范

### 6.1 命名约定

| 元素 | 规范 | 示例 |
|------|------|------|
| 类名 | PascalCase | `PrinterProfile` |
| 方法名 | PascalCase | `GetPrinters()` |
| 属性 | PascalCase | `OutputDirectory` |
| 私有字段 | _camelCase | `_ghostScriptPath` |
| 常量 | PascalCase | `MaxRetryCount` |
| 事件 | PascalCase + 前缀 On | `OnJobCompleted` |

### 6.2 MVVM 模式规范

```csharp
// ViewModel 属性 - 使用 INotifyPropertyChanged
public class MainViewModel : INotifyPropertyChanged
{
    private string _statusText;
    public string StatusText
    {
        get => _statusText;
        set { _statusText = value; OnPropertyChanged(); }
    }

    // 命令 - 使用 RelayCommand 模式
    public ICommand StartMonitorCommand { get; }
}
```

### 6.3 异步编程

- 所有 I/O 操作使用 `async/await` 模式
- 避免在 UI 线程上执行阻塞操作
- GhostScript 进程调用使用 `Process.Start()` + 异步等待

### 6.4 错误处理

```csharp
// 服务层统一 try-catch 模式
try
{
    // 业务逻辑
}
catch (GhostScriptException ex)
{
    _logService.Error($"GS转换失败: {ex.Message}");
    throw;
}
catch (Exception ex)
{
    _logService.Error($"未预期错误: {ex.Message}");
    throw;
}
```

### 6.5 注释规范

```csharp
/// <summary>
/// 监控打印池目录并处理新的 .prn 文件
/// </summary>
/// <param name="spoolPath">打印池目录路径</param>
```

## 8. 数据流 - 打印池监控管线

```
用户打印文档
      │
      ▼
Windows 打印服务
      │
      ▼
生成 .prn 文件到打印池目录
      │
      ▼
┌──────────────────┐
│  SpoolMonitor    │  FileSystemWatcher 检测新 .prn 文件
│  (Created 事件)  │
└────────┬─────────┘
         │
         ▼
┌──────────────────┐
│ 文件名解析       │  FileNameResolver 解析模板占位符
│ 生成目标路径     │  ConflictResolver 处理文件冲突
└────────┬─────────┘
         │
         ▼
┌──────────────────┐     需要水印?
│ 水印注入         │──── 是 ──▶ PsWatermarkEngine 注入 PS 代码
│ (可选步骤)       │              │
└────────┬─────────┘     否 ────┘
         │
         ▼
┌──────────────────┐
│ GhostScript      │  调用 gswin64c.exe 转换 PS → PDF
│ 转换引擎         │  GhostScriptService 执行
└────────┬─────────┘
         │
         ▼
┌──────────────────┐
│ 输出到目标目录   │  PDF 文件保存到配置的输出路径
│ 记录作业历史     │  JobHistoryService 写入记录
└────────┬─────────┘
         │
         ▼
    完成 / 错误处理
```

## 9. 扩展点

### 9.1 自定义水印处理器

`PsWatermarkEngine` 设计为可扩展，可通过继承或接口添加新的水印类型：

```csharp
// 水印配置已包含以下可扩展项:
// - TextWatermark: 文字水印 (字体、大小、颜色、角度、透明度)
// - ImageWatermark: 图片印章 (路径、位置、缩放、透明度)
// - PageOverlay: 页眉/页脚 (文本、页码、日期)
```

### 9.2 添加新的文件冲突策略

`ConflictResolver` 可通过添加新的处理方法扩展：

```csharp
public enum ConflictStrategy
{
    Overwrite,   // 覆盖已有文件
    Increment,   // 文件名递增 (如 file_1.pdf, file_2.pdf)
    Backup       // 备份已有文件后保存新文件
}
```

### 9.3 添加新的打印机管理后端

`PrinterManager` 当前使用 PowerShell/WMI，可替换为其他后端：

```csharp
// 当前实现: PowerShell + WMI

```

### 9.4 事件系统

各服务暴露事件供扩展：

